Acoustic Server Installation: AWS Setup

Kevin Weng

Note:The Amazon setup guides are here: https://aws.amazon.com/documentation/ if you need further reference.

  1. Set up an account at http://aws.amazon.com/. You have to enter a credit card, but you can create a micro-instance, and run it for a year for free.
  2. Launch the AWS Management Console.
  3. Sign in.
    You'll see this:
  4. Click on "EC2".
  5. Click on "Launch Instance".
  6. Click "Quick launch wizard".
  7. Select "Ubuntu Server 12.04.2 LTS for Cluster Instances".
  8. Click "download".
  9. This is a permission file that you will need to upload on every use of AWS. Save in a safe place and don't lose it. For this example, our file is located at:
    "/Users/kweng/data/PFRPLAB/projects/acoustic_geolocation_tool/AWS_ubuntu/Kevin_ubuntu_1.pem".
  10. Click "continue".
  11. You'll see:
  12. Click "Launch".
  13. Click "Close".
  14. You need to set the security group, so take note of the name of the security group for this instance. In this example it is "quicklaunch-1". Look for this at the bottom of the screen (scroll down in the section at the bottom pane).
  15. On the left side, click on "security groups".
  16. Click on the box for the line with the appropriate name. Go to bottom pane, click "inbound" tab.
  17. Create a new rule, pull down and select "HTTP".
  18. Hit "add rule" and then "apply rule changes".
  19. Then you should see (note that a new line has appeared under TPC Port (Service):
  20. Click "Elastic IPs".
  21. Click "Allocate new address".
  22. Click "Yes, allocate".
  23. Now go to top section of page, right click on light blue line with your IP address, and hit "associate".
  24. Select the instance (there should only be one, unless you have installed other instances on your AWS).
  25. Click "Yes, associate".
  26. Copy the IP address you see here (ours is 54.213.157.76):
  27. Click "instances" on left pane.
  28. Right click on the light blue line and hit "connect".
  29. You'll see:
  30. Find where you put the permission file and copy the path. Paste it into the "private key path" box. You may wish to click "save key location". Doing so saves the path so you don't have to copy paste in the future.
  31. Click "Launch SSH Client".
  32. Allow Java to run.
  33. If you are using a mac you might see this (Allow it):
  34. Read and accept the EULA.
  35. Several prompts will appear, Click Accept/Allow/Yes in all of them.
  36. If successful you'll get a command line box:
  37. Your AWS setup is now complete.
    Follow the directions for Installing A Local Instance to setup the Acoustic Server.

NOTE:
DO NOT CLOSE THE SSH TERMINAL WINDOW! Doing so will dramatically slow down the server on AWS.
